## 1. AI 辅助后端开发

### 1.1 适合 AI 辅助的后端任务

**什么任务适合 AI 编程？**
- **样板代码生成**：CRUD 模板、DTO 转换、工具类
- **API 调用示例**：调用第三方 API（支付、短信、地图）的 SDK 使用
- **代码重构**：提取方法、拆分大类、命名优化
- **Bug 修复**：根据报错信息生成修复建议
- **测试用例生成**：单元测试、集成测试用例
- **代码解释**：复杂逻辑的注释生成
- **文档生成**：API 文档、README 注释
- **设计模式应用**：将一段代码改写为设计模式实现

### 1.2 不适合 AI 辅助的后端任务

**什么任务不适合 AI 编程？**
- **复杂业务逻辑设计**：需要深入理解业务规则和边界情况
- **系统架构决策**：技术选型、模块划分、依赖管理
- **性能优化**：需要实际压测数据支撑
- **安全性敏感代码**：认证授权、密码处理、加密解密
- **新技术/框架的第一次使用**：缺乏训练数据，容易出错
- **数据库迁移**：Schema 变更需要精确控制

### 1.3 AI 辅助代码审查

**如何使用 AI 做代码审查？**
1. **提供完整上下文**：代码 diff、相关测试结果
2. **明确审查重点**：性能、安全、可读性、异常处理
3. **分阶段审查**：先看业务逻辑，再看技术实现

**AI 代码审查的优势**
- 发现浅层问题：命名不规范、重复代码、缺少空检查
- 快速理解陌生代码
- 提供改进建议和示例代码

**AI 代码审查的局限**
- 复杂业务逻辑仍需人工判断
- 无法发现运行时问题（空指针、并发问题）
- 对项目的整体架构理解有限

## 5. 多级缓存架构与选型（追问链）

> 当候选人提到"多级缓存"时，必须追问选型理由和场景适配。

### 5.1 本地缓存（进程内缓存）

**适用场景：**
- 数据访问频率极高（如基础配置数据、热点省份城市数据）
- 对延迟极其敏感（<1ms 级别，Redis 即使是集群版也要 1-5ms）
- 数据几乎不变（变更频率极低，不需要考虑一致性问题）
- 单机部署或可以接受多实例数据不一致

**不适用场景（应有反问）：**
- 大数据量 → 本地缓存占用 JVM 堆内存，内存浪费严重
- 多实例部署 → 各实例缓存独立，数据不一致难以维护
- 频繁变更数据 → 缓存穿透问题，更新时需要同时更新所有实例
- 需要精确一致性的数据 → 本地缓存天然不具备一致性保证

**常见本地缓存实现：**
- Caffeine（Java 高性能缓存，推荐）
- Guava Cache
- ConcurrentHashMap + ScheduledExecutorService（手动实现）

### 5.2 Redis（分布式缓存）

**适用场景：**
- 需要跨进程/跨机器共享数据
- 数据有更新需要一致性
- 大数据量（不受单 JVM 堆内存限制）
- 需要缓存集群的高可用

**Redis 并不存在大量网络开销问题：**
- Redis QPS 可达 10万+，延迟可达亚毫秒级
- 网络开销不是 Redis 的核心瓶颈
- 本地缓存的主要目的不是"减少网络开销"，而是**极致的读取延迟**

### 5.3 本地缓存 + Redis 组合（L1 + L2）

**典型模式：**
```
请求 → 本地缓存(L1) → Redis(L2) → DB
命中 L1：延迟 < 1μs（纳秒级）
命中 L2：延迟 1-5ms
穿透 L1+L2：延迟 10-50ms
```

**什么时候需要 L1 + L2？**
- 极高频访问（每秒数万次）的热点数据
- Redis 集群仍然成为瓶颈的场景
- 愿意为极致的读取性能付出维护复杂度的代价

### 5.4 追问链（必须连续追问）

1. **第一层：你为什么要用本地缓存？**
   - 如果只说"减少网络开销" → 质疑：Redis 网络开销不是问题

2. **第二层：你的数据量多大？占多少内存？**
   - 如果数据量大 → 质疑：JVM 堆内存够用吗？不怕 GC 压力吗？

3. **第三层：多实例部署时本地缓存怎么同步？**
   - 如果没考虑 → 指出这是架构缺陷

4. **第四层：真的需要这么极致的性能吗？Redis 不够用吗？**
   - 如果没有充分理由 → 质疑是否过度设计

### 5.5 常见错误回答模式

| 错误回答 | 面试官质疑 |
|---------|----------|
| "减少网络开销" | Redis 网络开销很小，你的瓶颈真的在这吗？ |
| "数据量不大" | 那为什么不直接用 Redis？引入本地缓存增加了复杂度 |
| "我觉得这样快" | 有没有实际压测数据支撑？Redis 多少 ms？本地缓存多少 ms？ |
| "大家都这么用" | 社区常用不代表适合你的场景 |
