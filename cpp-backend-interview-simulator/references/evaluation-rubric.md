# 评分标准与反馈模板

## 评分维度

| 维度 | 权重（实习） | 权重（应届） | 权重（社招） |
|------|-------------|-------------|-------------|
| C++ 技术深度 | 25% | 30% | 35% |
| 项目经验 | 25% | 30% | 35% |
| 思维逻辑 | 25% | 20% | 15% |
| 表达能力 | 15% | 10% | 8% |
| 学习能力 | 10% | 10% | 7% |

> 社招权重偏向技术深度和项目经验（生产级），实习/应届权重偏向基础和学习潜力。

## 各风格反馈措辞示例

同一份面试结果，六种风格会给出不同的反馈措辞。以下是社招应届生同一场面试后的反馈对比：

### 严厉拷打型

```
综合评分 5.8/10。

C++ 技术深度 5.0：虚函数表结构能说清楚，问到移动语义就只停在"std::move 就是转右值"，
没有深入到引用折叠和完美转发。不足。

项目经验 6.0：项目规模偏小，QPS、数据量都说不出来，线上故障经验为零。

思维逻辑 6.0：还行，基本能讲清自己做了什么。

表达能力 6.0：基本合格。

学习能力 5.0：Copilot 在用，但说不出 deeper 的用法。

你现在出去面大概率挂。主要问题是项目深度不够 + 现代 C++ 特性理解太浅。建议：
1. 把《Effective Modern C++》读一遍，尤其是移动语义和完美转发章节
2. 整理一个项目中的性能数据，哪怕是压测数据也好
3. 上面两条没做到之前别投了。
```

### 温和鼓励型

```
综合评分 5.8/10，整体表现中规中矩，有一些亮点也有一些需要提升的地方。

C++ 技术深度 5.0：C++ 基础还是比较扎实的，虚函数原理能讲清楚，继续保持。
如果能再深入一些移动语义和完美转发就更好了。

项目经验 6.0：你有一个完整的项目经历，这在应届生中是不错的。
建议接下来可以更深入地了解项目的技术细节，特别是高性能并发相关的实现。

思维逻辑 6.0：你的逻辑思维很不错，说话有条理，这点在面试中很重要。

表达能力 6.0：表达清晰，能把自己的想法准确传达出来，继续保持。

学习能力 5.0：看得出来你有在用 AI 工具辅助学习，这是个好的开始。

建议下一步：1. 深入学习现代 C++ 特性（Effective Modern C++）；2. 整理项目中的技术难点和解决方案。
总体来说基础不错，继续加油！
```

### 专业高效型

```
综合评分 5.8/10。

【结论】C++ 技术深度基本合格，项目深度待加强，思维逻辑良好。

C++ 技术深度 5.0/10 — 虚函数 5 分，移动语义 4 分（引用折叠不熟）
项目经验 6.0/10 — 有项目经历，规模偏小，缺乏数据支撑
思维逻辑 6.0/10 — 逻辑清晰，表达有条理
表达能力 6.0/10 — 准确，无废话
学习能力 5.0/10 — AI 工具使用较浅

【核心短板】项目规模偏小，缺少生产级性能数据；现代 C++ 特性深度不足。

【下一步建议】
  1. 精读《Effective Modern C++》，重点理解移动语义、完美转发、右值引用
  2. 用压测数据量化项目性能指标（QPS、延迟）
  3. 系统学习分布式系统场景题
```

### 深挖学术型

```
综合评分 5.8/10。

你的思维路径比较清晰，但对一些核心机制的理解还停留在"描述性"阶段，
缺乏"分析性"视角。这是应届生常见的问题，需要从"知道是什么"升级到"理解为什么"。

C++ 技术深度 5.0：你描述了虚函数表的结构，但没有进一步分析为什么编译器选择这种布局、
多继承下 vtable 如何处理偏移。这些问题能考察你对编译器实现的深层理解。

项目经验 6.0：项目完整，但缺少对技术选型的深度复盘。

思维逻辑 6.0：论证路径基本清晰，但部分环节跳跃。

学习能力 5.0：AI 工具使用案例比较浅，建议了解底层代码生成原理。

建议下一步：读 C++ ABI 规范中关于虚函数布局的章节，以及 Scott Meyers 关于移动语义的原始演讲。
```

### 工程实践型

```
综合评分 5.8/10。

你的优势是有完整的项目经历，这是很多应届生缺乏的。但项目的技术深度不够。

C++ 技术深度 5.0：能讲清楚基本原理，但结合生产场景就卡住了。
比如你说用了 shared_ptr，但我问你有没有遇到过循环引用、线上有没有内存泄漏排查经验，
你说"应该没有吧"——这就是没有复盘的表现。

项目经验 6.0：项目在跑，这很好。但我需要数据：QPS 多少？峰值多少？
慢接口是哪几个？怎么用 perf/gdb 排查的？这些说不出来，在生产环境会很被动。

思维逻辑 6.0：基本逻辑是清晰的。

建议下一步：
1. 回顾项目中的三个核心接口，测出 QPS 和延迟数据
2. 给自己项目做一次全链路压测，整理成文档
3. 去了解一下生产环境的监控和 profiling 工具（perf、gdb、火焰图）
```

### 综合平衡型

```
综合评分 5.8/10。

今天的面试表现中规中矩，有一些做得不错的地方，也有明显的提升空间。

C++ 技术深度 5.0：C++ 基础是达标的，虚函数、智能指针都能讲清楚。
不过移动语义和完美转发是薄弱点，建议重点看一下。

项目经验 6.0：你有实际参与项目的经历，这是加分项。
不过项目的技术细节还需要更深入，比如能说出具体的性能数据和技术选型的权衡。

思维逻辑 6.0：思路比较清晰，能有条理地表达自己的观点。

表达能力 6.0：表达流畅，没有太多废话，这点很好。

学习能力 5.0：AI 工具在用，建议深入了解一些高级用法。

下一步建议：
1. 深入移动语义和完美转发，重点理解引用折叠规则
2. 整理项目中的性能指标和故障排查案例
3. 整体学习路线推荐：《Effective Modern C++》+ 《C++ Concurrency in Action》

今天的面试是一次很好的练习，继续准备，下次会更好。
```

## 面试进行中提示（不透露分数，只给引导）

### 严厉拷打型

| 场景 | 话术 |
|------|------|
| 回答模糊 | "不够。说到哪算哪？重来。" / "你说的这是背的，不是理解的。" |
| 遇到困难 | "这都不知道？那你怎么敢投这个岗位的？" |
| 回答较好 | "行了，这个点过了。" / "这种程度还不够，继续。" |
| 压力测试 | "不够。如果你写的高并发服务突然 CPU 打满，你打算怎么排查？" |

### 温和鼓励型

| 场景 | 话术 |
|------|------|
| 回答模糊 | "我理解你的意思。能再具体说说吗？比如……" |
| 遇到困难 | "没关系，这个问题确实有点偏。我给你一点提示……" |
| 回答较好 | "很好！这个问题你理解得很到位。能再补充一下吗？" |
| 压力测试 | "好，那我们来想想另一种情况……"（不说"错了"，而是说"那另一种思路呢？"）|

### 专业高效型

| 场景 | 话术 |
|------|------|
| 回答模糊 | "这个回答没有覆盖核心点。我们换一个问法……" |
| 遇到困难 | "好，这个点跳过。基础部分继续。"（不等，不追问）|
| 回答较好 | "好，这个点清楚了。项目深挖继续。" |
| 压力测试 | "我们来讨论一个边界情况：如果这段代码在异常发生时，你能保证资源不泄漏吗？" |

### 深挖学术型

| 场景 | 话术 |
|------|------|
| 回答模糊 | "我不太清楚你的论证路径。能先告诉我，你为什么这么认为吗？" |
| 遇到困难 | "这个问题没有标准答案。我更想知道你基于什么假设得出这个结论？" |
| 回答较好 | "好，那我们来延伸一下——这个问题在 C++ 标准委员会的讨论里是怎么考虑的？" |
| 压力测试 | "好，你给了方案 A，那有没有人提出过方案 B？他们的核心分歧在哪里？" |

### 工程实践型

| 场景 | 话术 |
|------|------|
| 回答模糊 | "你们实际上线的时候怎么部署的？用的什么机器规格？" |
| 遇到困难 | "这个问题我在生产环境遇到过，大概是这么个情况……你想想你会怎么排查？" |
| 回答较好 | "好，那你们有没有遇到过内存踩界的 core dump？怎么定位的？" |
| 压力测试 | "你说你做过性能优化，那你调优前后的 QPS 和延迟数据是什么？" |

### 综合平衡型

| 场景 | 话术 |
|------|------|
| 回答模糊 | "你说的方向是对的。能再具体一点吗？比如你在这个点具体做了什么？" |
| 遇到困难 | "没问题，这个点有难度。我换一种问法……你觉得这个和 XX 有什么关系？" |
| 回答较好 | "很好，这个点理解到位了。顺着这个我想多问一层……" |
| 压力测试 | "好，那我们延伸一下：如果场景变成……你会怎么考虑？"（讨论式，不说"不够"）|

### 通用（所有风格通用）

| 场景 | 话术 |
|------|------|
| 时间紧张时 | "由于时间关系，这个问题先到这里。" |
| 连续两次答不上来 | "提示一下：这个问题涉及 XX 原理/机制。"（给出一个方向的提示，不直接给答案）|

```
## E. 追问深度评估标准（新增）

> 本节定义"追问深度是否足够"的标准，用于评估候选人是否真正深入理解技术原理。

### E.1 连续追问判定

**深度不足（问一个点就过）：**
- ❌ 问虚函数原理，答完就过
- ❌ 问智能指针，说个引用计数就跳下一题
- ❌ 问 epoll，说个 IO 多路复用就结束

**深度足够（连续追问 3-5 层）：**
- ✅ 虚函数 → vtable 内存布局 → 多继承下 vtable → 虚函数调用开销 → devirtualization 优化
- ✅ 智能指针 → shared_ptr 引用计数 → weak_ptr 解决循环引用 → enable_shared_from_this → 自定义删除器
- ✅ epoll → ET/LT 模式区别 → 为什么 ET 要非阻塞 → epoll 惊群 → io_uring 对比

**面试官追问链检查清单：**
- [ ] 每个技术点是否追问了 3 层以上？
- [ ] 下一题是否基于上一题回答延伸？（不是换新话题）
- [ ] 如果候选人答错，是否追质疑而非跳过？

### E.2 新增考点评估

以下考点需纳入 C++ 面试评估：

**1. 现代 C++ 掌握程度（C++11/14/17/20）**

| 等级 | 表现 |
|------|------|
| 优秀 | 能熟练使用 C++17/20 特性，清楚各版本的演进动机 |
| 合格 | 掌握 C++11 核心特性，了解 C++14/17 部分特性 |
| 不合格 | 只会 C++98/03，不知道 move 语义、lambda 等 |

**2. 内存安全意识**

| 等级 | 表现 |
|------|------|
| 优秀 | 能系统性地避免内存泄漏、use-after-free、double-free，熟练使用 AddressSanitizer/Valgrind |
| 合格 | 了解 RAII 和智能指针，知道常见内存问题 |
| 不合格 | 不关心内存管理，写代码不考虑资源生命周期 |

**3. 性能优化意识**

| 等级 | 表现 |
|------|------|
| 优秀 | 能说出具体量化指标，有 profiling 经验，了解 cache line/分支预测/SIMD |
| 合格 | 知道需要量化验证，但方法不系统 |
| 不合格 | 只说"体感快了"、"大家都这么写" |

──────────────────────────────────────────
```

**匹配度评级标准：**

| 星级 | 含义 | 说明 |
|------|------|------|
| ★★★★★（5/5） | 高度匹配 | 技术栈、经验、能力全面覆盖 JD 要求，面试表现优异 |
| ★★★★☆（4/5） | 较好匹配 | 主要要求匹配，有 1-2 个小缺口但可短期补齐 |
| ★★★☆☆（3/5） | 基本匹配 | 核心技能匹配但有明显缺口，需要重点准备 |
| ★★☆☆☆（2/5） | 匹配不足 | 多个关键要求不满足，需要较长时间准备 |
| ★☆☆☆☆（1/5） | 严重不匹配 | JD 要求与当前能力差距很大，建议调整目标岗位 |
